baseUrl: "." を指定すると、プロジェクトルート（tsconfig.json があるディレクトリ）を基準にした絶対パスのようなインポートが使えるようになります。 // 通常の相対パス import prisma from '../../../lib/prisma' // baseUrl と paths を使えば import prisma from '@/lib/prisma' もし baseUrl が設定されていないと、"paths" は無視されてしまう つまり、paths とセットで baseUrl は必須です。 ============================================================================= ※これが@/を先頭に付ければtsconfig.jsonの位置を基準とした相対パスのように使える設定 { "compilerOptions": { "baseUrl": ".", // ここが必須！ "paths": { "@/*": ["./*"] }, ... } } ============================================================================= baseUrl: "." は tsconfig.json があるディレクトリを基準にする設定なので、 tsconfig.json が プロジェクトルート直下にある場合、 baseUrl: "." は「プロジェクトルート」を意味します。 そのため、 "@/*": ["./*"] の ./* は「プロジェクトルートからの相対パス」ということになります。 なので、もし tsconfig.json を src/ フォルダ内に置いたら、@/ は src/ を起点にしたパスになります。 通常の Next.js プロジェクトは tsconfig.json はルート直下にあるので、 @/ は「プロジェクトルートからの絶対パスのように使える」 ========================================================================== もし baseUrl がなくてもエラーにならない場合は？ TypeScript のバージョンやIDEの挙動、Next.jsの内部処理によっては paths が動作しているように見える場合もあります。 ただしこれは 公式仕様的には保証されていないので、 エラーにならないのは「ラッキー」か、何か別の要因で解決されている可能性があります。 安全に確実に使うなら必ず baseUrl を指定しましょう。 書かない場合は環境によって動いたり動かなかったりするので、トラブルの元になります。 ==========================================================================
